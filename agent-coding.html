AI Agent Use in Software Development

Practical tips for leveraging coding agents

The Paradigm Shift: Vibe Coding & Agentic AI

Vibe Coding Defined

  • "An approach to producing software by using artificial intelligence (AI), where a person describes a problem in a few sentences as a prompt to a large language model (LLM) tuned for coding."
  • Shift: Programmer's role moves from manual coding to guiding, testing, and refining AI-generated code.

Source: Ran Isenberg (quoting Wikipedia)

AI Mirrors Human Work Processes

  • Time: "More time, better results – Like humans, AI performs better when given time to think."
  • Tools: "Using tools – AI becomes more capable with better tools at its disposal — just like humans do."
  • Feedback: "Learning from feedback – AI improves through feedback loop, much like humans learn from experience."

Source: Augment Code

Detail & Specificity

  • "Prompts should be detailed and not too short."
  • "Be precise about the task. Include both functional and non-functional requirements."
  • "Specificity leads to better alignment with expectations."

Sources: Augment Code, Isenberg, Claude Code

Provide Context (Crucial!)

  • Problem Domain: "Help the agent understand the overall problem domain and his expertise in this domain."
  • Reasoning & Constraints: "Explain not only the final goal but also reasoning behind, additional constraints, etc."
  • External Docs: Share "relevant documentation such as tickets, GitHub issues, or PRs."

Sources: Isenberg, Augment Code

Internal Context (MCP & CLAUDE.md)

  • MCP (Model Context Protocol): Integrate internal sources (Jira, Confluence, Notion, GitHub) via MCP servers.
    • "The tighter the integration, the smarter your agent becomes."
  • CLAUDE.md: Files for automatic context pulling (e.g., bash commands, style guides, testing instructions).

Sources: Isenberg, Claude Code

Prompting Techniques

  • Define Persona: "Start by setting the stage." E.g., "You're a senior AWS Serverless Python developer..."
  • Ask for a Plan First: For complex tasks, "Before it starts coding, ask it to explain its plan."

Sources: Augment Code, Isenberg, Claude Code

Workflow Strategies & Best Practices

Break Down Complex Tasks

  • "Break complex tasks into smaller, digestible pieces (one at a time)."
  • "If the prompt gets too long... break it into smaller prompts that build upon the changes of the previous prompt." (Chained Prompting)

Sources: Augment Code, Isenberg

Iterative Development & Feedback Loops

  • Leverage Test Results: "Agent excels at iterating on test results and code execution output."
  • Ask agent to "add tests," "run them," and "parse the test output" to fix problems.
  • Visual Feedback: Provide "visual targets" (browser screenshots, mocks) and iterate until design matches.

Sources: Augment Code, Isenberg, Claude Code

Understanding What You Commit

  • "Never ship code you don't fully understand."
  • Always "review, refactor, and test everything it produces."

Source: Ran Isenberg
